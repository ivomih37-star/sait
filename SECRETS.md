# Секреты, ключи и логи — единое хранилище

Как в этом проекте организовано хранение чувствительных данных (токены,
API-ключи, логи), чтобы Claude Code мог их читать локально, но они никогда
не попадали в git и не публиковались.

> ⚠️ Это репозиторий (публикуется на GitHub Pages). Ничего из этого раздела
> не должно оказаться в коммите. Реальные значения ключей — только на вашей
> машине или в GitHub Secrets (для CI), никогда в файлах, которые видит git.

---

## Где что лежит

Одна папка на корне репозитория — `.secrets/` (не коммитится, см. `.gitignore`):

```
.secrets/
├── .env          # все токены и ключи, формат KEY=VALUE
└── logs/         # логи локальных запусков (бот, скрипты и т.п.)
```

- `.secrets/.env` — единый файл со всеми секретами проекта (Telegram-токен,
  сторонние API-ключи и т.д.), а не россыпь `.env` в разных подпапках.
- `.secrets/logs/` — сюда пишут логи локальные скрипты (`telegram/bot/bot.py`
  и другие), чтобы Claude Code мог их прочитать при отладке без доступа к
  вашему терминалу.

Папка целиком в `.gitignore` — она никогда не попадёт в git, даже если внутри
появятся новые файлы.

## Как этим пользуется Claude Code

- Claude Code (локальный CLI на вашей машине) имеет доступ к файловой системе
  проекта, включая игнорируемые git файлы — `.gitignore` скрывает файлы только
  от git, не от чтения диском. Достаточно попросить: «глянь токен в
  `.secrets/.env`» или «посмотри последний лог в `.secrets/logs/`».
- В **удалённой (cloud/web) сессии** Claude Code эта папка обычно пустая —
  контейнер эфемерный и не видит вашу локальную машину. Секреты для такой
  сессии кладите как **GitHub Actions Secrets** (Settings → Secrets and
  variables → Actions) — см. пример в `TELEGRAM_SETUP.md`, шаг 4.

## Как завести секрет

1. Создать (один раз) `.secrets/.env` в корне репозитория.
2. Добавить строку `KEY=значение` — например:
   ```
   TELEGRAM_BOT_TOKEN=123456789:AA...
   ```
3. Для локального бота (`telegram/bot/`) можно продолжать использовать его
   собственный `.env` (см. `telegram/bot/.env.example`) — он тоже в
   `.gitignore`. `.secrets/.env` — для всего остального (новые интеграции,
   ключи для скриптов вне `telegram/bot/`).

## Правила

- Никогда не вставлять значения токенов/ключей в чат, коммиты, issue или PR —
  только имя переменной (`TELEGRAM_BOT_TOKEN`), не значение.
- Не логировать секреты в `.secrets/logs/` — только обычные логи выполнения.
- Если секрет случайно попал в git-историю — считать его скомпрометированным
  и немедленно перевыпустить (для Telegram-бота — `/revoke` у @BotFather).
