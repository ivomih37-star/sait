# Telegram-канал «Ракия Клуб» — создание и автоматизация

Пошаговый план: завести канал, подключить бота и наладить автопубликацию по
расписанию из этого репозитория (GitHub Actions). Роль агента — готовить и
ставить посты в очередь; публикует их бот автоматически.

> 🔒 **Безопасность.** Токен бота — это пароль. Никогда не вставляйте его в чат,
> в README или в код. Хранить только как **GitHub Secret** (см. шаг 4).

---

## Архитектура (рекомендуемая)

```
Репозиторий (контент)              GitHub Actions (по расписанию)        Telegram
┌────────────────────┐             ┌───────────────────────────┐        ┌──────────┐
│ telegram/queue.json│  ──читает── │ publish.py + cron-workflow │ ─API─▶ │  Канал   │
│ (дата, текст, фото)│             │ берёт пост на сегодня      │        │ @raki... │
└────────────────────┘             └───────────────────────────┘        └──────────┘
        ▲                                      │
        │ редактирую я (агент) / ты            │ отмечает «отправлено» коммитом назад
```

- **Бесплатно**, ничего не нужно держать включённым.
- Контент — в репозитории, значит правки идут через привычные коммиты/PR.
- Альтернативы (если не хочется Actions): Python-скрипт у себя по cron, no-code
  (n8n/Make), либо вообще без автоматики — нативные «отложенные сообщения»
  Telegram (см. раздел «План Б»).

## Шаг 1. Создать канал

1. Telegram → меню **☰ → Новый канал**.
2. Название: **Ракия Клуб · Москва**. Описание — короткое из `BRAND.md`.
3. Тип: **Публичный**, задать адрес-юзернейм (напр. `@raki_club_msk`).
   - Публичный нужен для удобной ссылки и простого `chat_id` (`@raki_club_msk`).
4. Аватар: можно взять плашку из `post-images/` или флаг 🇧🇬.
5. Закреп позже — манифест из `MARKETING.md` (пост №1).

## Шаг 2. Создать бота

1. Напишите **@BotFather** → команда `/newbot`.
2. Имя бота: `Ракия Клуб Постер`. Юзернейм: `raki_club_poster_bot` (должен
   оканчиваться на `bot`).
3. BotFather выдаст **токен** вида `123456789:AAE...`. Скопируйте его в надёжное
   место (менеджер паролей). В чат не присылать.

## Шаг 3. Сделать бота админом канала

1. Канал → **Управление → Администраторы → Добавить**.
2. Найдите бота по юзернейму, выдайте право **«Публикация сообщений»**
   (Post Messages). Остальные права не нужны.

## Шаг 4. Сохранить секреты в GitHub

Репозиторий → **Settings → Secrets and variables → Actions → New repository
secret**. Создать два секрета:

| Имя | Значение |
|---|---|
| `TELEGRAM_BOT_TOKEN` | токен из BotFather |
| `TELEGRAM_CHAT_ID` | `@raki_club_msk` (для публичного канала) |

> Для приватного канала `chat_id` числовой (вид `-100…`). Получить: добавить
> бота, отправить любой пост, открыть
> `https://api.telegram.org/bot<TOKEN>/getUpdates` и взять `chat.id`.

## Шаг 5. Очередь контента

Файл `telegram/queue.json` — массив постов. Пример одной записи:

```json
{
  "id": "01-manifest",
  "date": "2026-06-29",
  "text": "Наздраве, Москва! 🇧🇬🥃\n\nМы — клуб любителей болгарской ракии…",
  "image": "post-images/card-anons.png",
  "pin": true,
  "sent": false
}
```

- `date` — когда опубликовать (по календарю из `MARKETING.md`).
- `image` — путь к картинке в репо (необязательно).
- `pin` — закрепить после отправки (для манифеста).
- `sent` — ставится `true` после успешной отправки (коммитом назад).

Тексты уже готовы — переносим 16 постов из `MARKETING.md`.

## Шаг 6. Скрипт публикации

`telegram/publish.py` (псевдо-логика):

1. Прочитать `queue.json`.
2. Выбрать посты с `sent == false` и `date <= сегодня`.
3. Для каждого: `sendPhoto` (если есть `image`) или `sendMessage`
   (`parse_mode=HTML`, `disable_web_page_preview` по вкусу).
4. При `pin: true` — вызвать `pinChatMessage`.
5. Пометить `sent: true`, записать `queue.json`, закоммитить обратно.

Bot API, без зависимостей (только `urllib`):
`POST https://api.telegram.org/bot<TOKEN>/sendMessage?chat_id=…&text=…`

## Шаг 7. Workflow по расписанию

`.github/workflows/telegram.yml`:

- `on: schedule` — например `cron: "5 7 * * *"` (≈10:05 МСК; cron в Actions в
  **UTC** и может опаздывать на несколько минут — для дневных постов нормально).
- `on: workflow_dispatch` — кнопка «опубликовать сейчас» вручную.
- Шаги: checkout → `python telegram/publish.py` (с секретами в env) → commit
  обратно изменённый `queue.json` (нужен `permissions: contents: write`).

## Шаг 8. Тест

1. Добавить тестовый пост с сегодняшней датой.
2. Запустить workflow вручную (**Actions → Telegram → Run workflow**).
3. Проверить, что пост пришёл в канал и `sent` стал `true`.
4. Дальше — работает само по расписанию.

## Как это «через агента»

- **Я готовлю контент:** добавляю/редактирую посты в `queue.json`, проставляю
  даты, прикрепляю картинки из `post-images/`, оформляю по `BRAND.md`.
- **Публикует бот** из GitHub Actions по расписанию — меня не нужно держать
  включённым.
- **Срочно опубликовать:** говоришь мне — я добавляю пост датой «сегодня» и
  запускаю workflow вручную (`workflow_dispatch`).
- Прямая отправка из этой агент-сессии в Telegram, скорее всего, заблокирована
  сетевой политикой песочницы — поэтому канал публикации именно Actions.

## План Б — без кода

Если автоматизация не нужна прямо сейчас:

- **Отложенные сообщения Telegram.** В канале зажмите «отправить» → «Отправить
  позже» → выберите дату/время. Так можно заранее запланировать все 16 постов
  руками. Минус — нельзя массово и нет картинок-каруселей по расписанию.
- **No-code (n8n/Make):** триггер по времени → нода Telegram (token + chat_id) →
  текст. Быстро мышкой, но это сторонний сервис со своим хостингом.
