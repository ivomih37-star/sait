# Как создать Telegram-бота и привязать к проекту

Конкретные шаги: завести бота, дать ему права в канале и подключить к
автопостингу этого репозитория (`telegram/publish.py` + Actions).

> 🔒 Токен бота — это пароль. Только в GitHub Secrets. Никогда в чат и код.

---

## 1. Создать бота в @BotFather

1. Открыть в Telegram **@BotFather** → `/newbot`.
2. **Имя** (любое, видно людям): `Ракия Клуб Постер`.
3. **Юзернейм** (уникальный, заканчивается на `bot`): напр. `raki_club_poster_bot`.
4. BotFather пришлёт строку-токен вида `8123456789:AAH...xyz` — это
   `TELEGRAM_BOT_TOKEN`. Скопировать в менеджер паролей.

Полезные команды BotFather (по желанию):
- `/setuserpic` — аватар бота (можно плашку из `post-images/`).
- `/setdescription`, `/setabouttext` — описание.
- `/setprivacy` → **Disable**, если бот должен видеть сообщения в группах
  (для постинга в канал не обязательно).

## 2. Создать канал (если ещё нет)

Telegram → ☰ → **Новый канал** → название «Ракия Клуб · Москва» →
тип **Публичный**, юзернейм напр. `@raki_club_msk`.

## 3. Сделать бота админом канала

1. Канал → **Управление → Администраторы → Добавить администратора**.
2. Найти бота по юзернейму (`@raki_club_poster_bot`).
3. Включить право **«Публикация сообщений»** (Post Messages). Для закрепа
   манифеста — также **«Закрепление сообщений»** (Pin Messages). Сохранить.

Без этого шага бот получит ошибку `403: bot is not a member / not enough rights`.

## 4. Узнать chat_id канала

- **Публичный канал:** `chat_id` = `@raki_club_msk` — больше ничего не нужно.
- **Приватный канал** (числовой id):
  1. Опубликуйте любой пост в канал.
  2. Перешлите его боту **@userinfobot** (или @getidsbot) — он покажет id вида
     `-1001234567890`.
  3. Это и есть `TELEGRAM_CHAT_ID`.

### Быстрая проверка токена (curl)
На своём компьютере (не в этом чате):
```bash
curl "https://api.telegram.org/bot<ТОКЕН>/getMe"
# ответ {"ok":true,"result":{"username":"raki_club_poster_bot",...}} = токен валиден
```

## 5. Привязать к проекту: секреты в GitHub

Репозиторий **ivomih37-star/sait** → **Settings → Secrets and variables →
Actions → New repository secret**. Создать два:

| Имя секрета | Значение |
|---|---|
| `TELEGRAM_BOT_TOKEN` | токен из BotFather |
| `TELEGRAM_CHAT_ID` | `@raki_club_msk` или `-100…` |

Эти имена уже ожидает воркфлоу `.github/workflows/telegram.yml` — менять код не
нужно.

## 6. Тестовый прогон

1. GitHub → вкладка **Actions** → workflow **«Публикация в Telegram»**.
2. **Run workflow** → поле `dry_run` = `1` → Run. Это прогон без отправки:
   в логах видно, что и когда было бы опубликовано.
3. Если ок — запустить ещё раз с `dry_run` = `0` (или `пусто`). Посты, у которых
   дата ≤ сегодня и `sent:false`, уйдут в канал, а воркфлоу отметит их `sent`.

> Сейчас в очереди самая ранняя дата — 29.06.2026. Чтобы проверить «вживую»
> раньше, временно добавьте в `queue.json` тестовый пост с сегодняшней датой
> (или попросите агента) и запустите workflow.

## 7. Дальше — само

`schedule` в воркфлоу запускает публикацию ежедневно (~10:05 МСК). Каждый день
бот публикует посты, у которых наступила дата. Контент добавляется правкой
`telegram/queue.json` (через коммит/PR или через агента).

---

## Что делает бот в этом проекте

- **Только публикует** в канал по расписанию (он — «постер», не диалоговый бот).
- Источник постов — `telegram/queue.json` в этом репозитории.
- Запуск — GitHub Actions, токен подставляется из секрета, локально нигде не
  хранится.

## Если захотите диалогового бота (привет/правила новичкам)

Это отдельный сценарий: бот, который отвечает на команды в чате клуба
(`/start`, `/rules`, приветствие). Требует постоянно запущенного процесса
(webhook или long-polling) — это уже сервер/хостинг, не разовый workflow.
Скажите — распишу отдельно.

## Частые ошибки

| Симптом | Причина / решение |
|---|---|
| `401 Unauthorized` | Неверный токен. Проверьте `getMe`. |
| `400 chat not found` | Неверный `chat_id`. Для публичного — `@username` с `@`. |
| `403 not enough rights` | Бот не админ канала или нет права Post Messages. |
| Опрос не отправился | У бота нет права постить, либо канал приватный без id. |
| Картинка не пришла | Файл ещё не в ветке `main` (raw-URL пустой) — влейте PR. |
