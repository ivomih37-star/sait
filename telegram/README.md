# Telegram-автопостинг · Ракия Клуб · Москва

Автопубликация постов в канал по расписанию через GitHub Actions.
Полный план настройки — в [`../TELEGRAM_SETUP.md`](../TELEGRAM_SETUP.md).

## Файлы
- `queue.json` — очередь постов (дата, текст, картинка/опрос, флаги).
- `publish.py` — отправляет «созревшие» посты через Bot API (только stdlib).
- `../.github/workflows/telegram.yml` — запуск по cron + вручную.

## Что нужно один раз сделать
1. Создать канал и бота (BotFather), сделать бота админом канала.
2. Добавить секреты в GitHub → Settings → Secrets and variables → Actions:
   - `TELEGRAM_BOT_TOKEN` — токен бота
   - `TELEGRAM_CHAT_ID` — `@raki_club_msk` (или числовой `-100…`)
3. Готово: каждый день воркфлоу публикует посты, у которых наступила дата.

## Формат поста в queue.json
```json
{
  "id": "06-anons-vstrechi",   // уникальный id
  "date": "2026-07-08",         // когда публиковать (<= сегодня → отправится)
  "text": "Текст поста…",       // для опроса оставить ""
  "image": "post-images/card-anons.png",  // необязательно, путь в репо
  "poll": {                      // необязательно: вместо текста — опрос
    "question": "Вопрос?",
    "options": ["А", "Б", "В"]
  },
  "pin": true,                   // закрепить после отправки (для манифеста)
  "disable_preview": true,       // не разворачивать превью ссылок
  "sent": false                  // ставится true после публикации
}
```
- Картинки берутся по URL из репозитория (raw.githubusercontent…), поэтому
  достаточно указать путь — файл должен быть в ветке `main`.
- Опрос (`poll`) отправляется нативным голосованием Telegram.

## Локальная проверка (без отправки)
```bash
python telegram/publish.py          # DRY_RUN, если токен не задан
```
Покажет, что и когда было бы отправлено, ничего не публикуя.

## Ручной запуск / «опубликовать сейчас»
Actions → «Публикация в Telegram» → Run workflow (можно с `dry_run=1` для теста).

## Как добавить пост
Допишите объект в `queue.json` (или попросите агента) — с датой, текстом и при
необходимости картинкой/опросом. В нужный день он опубликуется сам.

⚠️ Токен — это пароль. Только в GitHub Secrets, никогда в код или чат. 18+.
