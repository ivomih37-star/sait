# Деплой диалогового бота на Railway

Railway собирает и запускает бота прямо из этого репозитория — Linux/SSH не
нужны. Бот работает как фоновый процесс (worker, long-polling).

> 🔒 Токен задаётся в переменных Railway, а не в коде. В git он не попадает.

## Что уже готово в репозитории
- `requirements.txt` — зависимость `python-telegram-bot`
- `Procfile` → `worker: python bot.py` (Railway запускает фоновый процесс)
- `.python-version` → `3.12`
- `bot.py` читает токен из переменной окружения `TELEGRAM_BOT_TOKEN`

## Шаг 1. Подготовка в Telegram (1 минута)
1. Добавь `@raki_club_helper_bot` в **чат клуба** (группа, не канал) и сделай
   **админом**.
2. @BotFather → `/mybots` → `@raki_club_helper_bot` → **Bot Settings → Group
   Privacy → Turn off**.

## Шаг 2. Создать проект на Railway
1. Зайди на **railway.app** → **Login with GitHub**.
2. **New Project → Deploy from GitHub repo** → выбери репозиторий
   **ivomih37-star/sait** (при необходимости дай Railway доступ к репозиторию).
3. Railway создаст сервис и попробует собрать — пока не важно, поправим в шаге 3.

## Шаг 3. Указать папку бота
1. Открой сервис → **Settings**.
2. **Root Directory** → впиши `telegram/bot` → сохрани.
   (Так Railway возьмёт `requirements.txt`, `Procfile` и `.python-version`
   именно из папки бота.)

## Шаг 4. Добавить токен
1. Вкладка **Variables → New Variable**.
2. **Name:** `TELEGRAM_BOT_TOKEN`
   **Value:** токен helper-бота от BotFather.
3. Сохрани — Railway автоматически передеплоит.

## Шаг 5. Проверить
1. Вкладка **Deployments → View logs**.
2. Должна появиться строка **`Бот запущен (long-polling).`** без ошибок.
3. Напиши боту `/start` в личке — ответит. Зайди тестовым аккаунтом в чат —
   придёт приветствие.

## Управление
- **Логи:** Deployments → View logs.
- **Перезапуск:** меню сервиса → Restart.
- **Обновление кода:** просто запушь изменения в `main` — Railway передеплоит сам.

## Стоимость
У Railway есть стартовый триал-кредит; дальше тариф Hobby (~$5/мес). Бот
потребляет мало — обычно укладывается в минимальную стоимость.

## Если что-то не так
- Лог пустой / падает сразу → проверь, что **Root Directory** = `telegram/bot`.
- `Не задан TELEGRAM_BOT_TOKEN` → добавь переменную (шаг 4), без пробелов.
- Бот не приветствует в чате → он должен быть **админом группы**, а Group
  Privacy — **выключен** (шаг 1). В **канале** приветствия не будет: Telegram
  не шлёт ботам события входа в каналы — только в группах.
