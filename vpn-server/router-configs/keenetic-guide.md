# Keenetic: WireGuard + split-tunnel по доменам

Инструкция для KeeneticOS 4.0+ (веб-интерфейс, без обязательного Entware — родная
поддержка WireGuard и политик маршрутизации по спискам доменов есть "из коробки"
начиная с этой версии прошивки).

## 1. Импорт конфига WireGuard

1. Скачайте `configs/router.conf` (создан `setup.sh`, режим полного туннеля).
2. В веб-интерфейсе Keenetic: **Интернет → Другие подключения → WireGuard → Добавить подключение**.
3. Выберите **«Импортировать из файла»** и укажите `router.conf`.
4. Сохраните — появится новое подключение `Wireguard0`. Дождитесь статуса «Подключено».
5. **Важно**: в свойствах подключения ОТКЛЮЧИТЕ опцию «Использовать для доступа в интернет»
   (или аналогичную «сделать основным шлюзом») — интернет по умолчанию должен идти как
   обычно, а через VPN пойдут только домены по правилу ниже. Если нужен полный туннель
   для всей сети — наоборот, включите её и пропустите шаг 3 ниже.

## 2. Списки доменов

Используйте файлы прямо с сервера — Keenetic принимает список доменов построчно:
- `routing/ru-domains.txt` — российские сайты (идут напрямую, без VPN)
- `routing/force-vpn-domains.txt` — сервисы с геоблоком (принудительно через VPN)

## 3. Политики маршрутизации (веб-интерфейс)

**Домашняя сеть → Политики маршрутизации** (или **Интернет → Политики маршрутизации**
в зависимости от версии прошивки):

1. **Создать политику** → назовите `VPN-force` → интерфейс `Wireguard0` →
   вкладка «Список доменов» → вставьте содержимое `force-vpn-domains.txt`
   (по одному домену на строку). Сохранить.
2. **Создать политику** → назовите `RU-direct` → интерфейс `ISP` (обычный WAN,
   тот что используется для доступа в интернет по умолчанию) → «Список доменов» →
   вставьте содержимое `ru-domains.txt`. Сохранить.
3. Примените обе политики к нужным устройствам/всей локальной сети в разделе
   **Приоритет политик** — `VPN-force` и `RU-direct` должны иметь более высокий
   приоритет, чем политика "по умолчанию" (если вы also настраиваете полный
   туннель через `Wireguard0` как основной шлюз).

## 4. Если нет политик маршрутизации по доменам в вашей прошивке

На старых версиях KeeneticOS (< 4.0) или моделях с урезанной прошивкой такой
функции в веб-интерфейсе может не быть. В этом случае:

1. Установите компонент **Entware** (Keenetic → Приложения → Entware, требует
   USB-накопитель или встроенную память с поддержкой).
2. Через Entware поставьте `dnsmasq-full` и `ipset` (`opk install dnsmasq-full ipset`).
3. Далее следуйте инструкции `openwrt-guide.md` — после установки Entware Keenetic
   получает практически тот же набор инструментов (uci/dnsmasq/ipset), что и OpenWrt.

## 5. Проверка

С устройства в локальной сети:
```
nslookup sberbank.ru      # обычный ISP-адрес, не VPN
nslookup netflix.com      # должен резолвиться и идти через Wireguard0
```
В веб-интерфейсе Keenetic: **Домашняя сеть → Устройства → (ваше устройство) →
статистика трафика по интерфейсам** покажет, через какой канал реально идёт трафик.

## 6. Обновление списков

При изменении `routing/ru-domains.txt` / `routing/force-vpn-domains.txt` на сервере —
повторно откройте политику маршрутизации в веб-интерфейсе Keenetic и обновите список
доменов (вставьте актуальное содержимое файла). Автоматической синхронизации с роутером
нет — это ручной шаг (веб-интерфейс Keenetic не имеет API для программной загрузки
списков без дополнительных надстроек).
