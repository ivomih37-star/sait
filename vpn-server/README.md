# VPN-сервер "в одну кнопку"

WireGuard VPN с веб-панелью управления, split-tunnel (российские сайты — напрямую,
остальное — через VPN), фолбэком Xray/VLESS+Reality на случай блокировки WireGuard,
DNS-over-HTTPS, автоматическими бэкапами и Telegram-уведомлениями.

Полная установка — одна команда на чистом Ubuntu 22.04 сервере, вводится только
пароль веб-панели.

## 1. Выбор и покупка сервера

| Провайдер | Тариф | Локация | Цена |
|---|---|---|---|
| **Hetzner** (рекомендуется) | CPX11 (2 vCPU / 2GB / 40GB NVMe) | Helsinki (Финляндия), Falkenstein/Nuremberg (Германия) | ~€4.35/мес |
| **Timeweb Cloud** | Минимальный VPS (1 CPU / 1GB / 15GB) | Нидерланды | ~250-350₽/мес, удобна оплата российской картой |
| **Vultr** | Regular Performance (1 vCPU / 1GB / 25GB) | Amsterdam, Frankfurt | ~$3.50-6/мес |

Локации Финляндия/Германия/Нидерланды дают минимальный пинг из России (обычно 30-60мс).

**При создании сервера укажите:**
- ОС: **Ubuntu 22.04 LTS** (x86_64 или ARM64 — оба поддерживаются)
- Добавьте свой SSH-публичный ключ (избегайте входа только по паролю)
- Тариф с 1-2 GB RAM, 10-20 GB SSD — этого достаточно с большим запасом

После создания сервера у вас будет: **IP-адрес**, пользователь (`root` или `ubuntu`),
и SSH-доступ (по ключу или временному паролю от провайдера).

## 2. Технологический стек

- **WireGuard** — VPN-протокол (быстрый, современный, малозаметен для DPI)
- **wg-easy** — веб-панель управления клиентами (Docker)
- **Xray (VLESS + Reality)** — фолбэк-протокол, маскируется под обычный HTTPS-трафик
  к реальному сайту, помогает при блокировке UDP/WireGuard по DPI
- **dnscrypt-proxy** — DNS-over-HTTPS, защита от утечек и подмены DNS
- **UFW + fail2ban** — файрвол и защита от перебора SSH
- **Docker Compose** — оркестрация всех сервисов

## 3. Установка (одна команда)

Подключитесь к серверу по SSH и выполните:

```sh
git clone <URL-этого-репозитория>
cd vpn-server
sudo ./setup.sh
```

Скрипт спросит:
1. **Пароль для веб-панели wg-easy** (придумайте свой, сохраните в надёжном месте)
2. **Telegram Bot Token / Chat ID** — по желанию, Enter чтобы пропустить (как настроить
   бота — см. [раздел 7](#7-мониторинг-и-обслуживание))

Дальше всё автоматически: обновление системы, установка Docker, запуск wg-easy/Xray/DNS,
настройка UFW/fail2ban/автообновлений, отключение IPv6, загрузка RU-списков, создание
трёх примеров клиентов (`client1-phone`, `client2-pc`, `router`).

В конце скрипт выведет команду для доступа к веб-панели и VLESS-ссылку для Xray.

**Неинтерактивная установка** (например, из CI/автоматизации):
```sh
WG_EASY_PANEL_PASSWORD='ваш-пароль' ./setup.sh
```

## 4. Split-tunnel: российские сайты напрямую

Реализовано на двух уровнях:

- **Телефон/ПК (WireGuard-клиент)**: технически WireGuard-клиенты умеют маршрутизацию
  только по IP-диапазонам (`AllowedIPs`), не по доменам. Поэтому используется список
  "весь интернет минус Россия" (`routing/world-minus-ru.txt`), который автоматически
  пересчитывается `scripts/update-routes.sh` (по расписанию — раз в неделю, из
  актуальных данных ipdeny.com / RIPE NCC). Российские сайты идут напрямую через
  обычный интернет, всё остальное — через VPN.
- **Роутер (Keenetic/OpenWrt)**: точная маршрутизация по конкретным доменам через
  dnsmasq+ipset — см. `router-configs/`. Списки доменов: `routing/ru-domains.txt`
  (напрямую) и `routing/force-vpn-domains.txt` (принудительно через VPN).

Списки доменов курируются вручную в `routing/*.txt` — дополняйте под свои нужды.
IP-диапазоны обновляются автоматически, вручную запустить: `./scripts/update-routes.sh`.

Честные технические ограничения этого подхода — см. `docs/SECURITY.md`.

## 5. Сервисы с геоблоком

Netflix, Spotify, Steam и подобные сервисы либо блокируют РФ по географии (значит нужен
VPN), либо детектируют и блокируют сам VPN по репутации IP дата-центра. Этот проект:

- Направляет трафик к таким сервисам через VPN (список — `routing/force-vpn-domains.txt`,
  используется роутерами; на телефоне/ПК они просто попадают в общий "мир минус РФ").
- Даёт Xray/VLESS+Reality как альтернативный транспорт с TLS-маскировкой — помогает
  против DPI-блокировок, но **не** против баз репутации IP (см. `docs/SECURITY.md`,
  раздел 1 — там же что делать, если ваш IP уже "спалился").

## 6. Подключение клиентов

### Добавление нового клиента

```sh
./scripts/add-client.sh мой-телефон           # split-tunnel (по умолчанию)
./scripts/add-client.sh мой-ноутбук --split
./scripts/add-client.sh домашний-роутер --full # полный туннель, для роутера
```
Появится QR-код в терминале + файлы `configs/<имя>.conf` и `configs/<имя>.png`.

### iPhone / Android
1. Установите приложение **WireGuard** (App Store / Google Play, официальное).
2. «+» → «Сканировать QR-код» → наведите камеру на `configs/<имя>.png`
   (просмотрите файл, например через `scp` на телефон, или отсканируйте прямо с
   экрана, если запускаете `add-client.sh` через терминал на телефоне/планшете).
3. Включите тумблер подключения.

### Windows / macOS
1. Скачайте официальный клиент **WireGuard** ([wireguard.com/install](https://www.wireguard.com/install/)).
2. «Импорт туннеля из файла» → выберите `configs/<имя>.conf`.
3. «Активировать».

### Роутер (Keenetic / OpenWrt)
Пошаговые инструкции — `router-configs/keenetic-guide.md` и `router-configs/openwrt-guide.md`.
Используйте конфиг, созданный с флагом `--full` (например `configs/router.conf`).

## 7. Мониторинг и обслуживание

- **Веб-панель** (добавление/удаление клиентов через браузер):
  ```sh
  ssh -L 51821:127.0.0.1:51821 <пользователь>@<ip-сервера>
  ```
  Затем откройте `http://127.0.0.1:51821` в браузере.

- **Telegram-уведомления**: создайте бота через [@BotFather](https://t.me/BotFather),
  получите токен, узнайте свой `chat_id` (например через [@userinfobot](https://t.me/userinfobot)),
  укажите оба значения при первом запуске `setup.sh` (или впишите в `.env` вручную и
  перезапустите `docker compose up -d`). Ежедневный статус сервера + уведомления
  об ошибках обновления списков/бэкапов приходят автоматически (systemd-таймер).

- **Бэкапы**: ежедневно автоматически (`systemd/vpn-backup.timer`) в `/root/vpn-backups/`,
  хранятся последние 14 копий. Вручную: `./scripts/backup.sh`. Восстановление:
  `./scripts/restore.sh /root/vpn-backups/vpn-backup-<дата>.tar.gz`.

- **Обновление RU-списков**: автоматически раз в неделю, вручную — `./scripts/update-routes.sh`.

## 8. Безопасность

- Веб-панель **не** доступна из интернета — только через SSH-туннель (см. выше).
- Смена порта SSH — **безопасно, в 2 шага** (не блокирует доступ):
  ```sh
  sudo ./scripts/harden-ssh.sh 2222          # добавляет 2222 параллельно с 22
  # проверьте в НОВОМ окне терминала: ssh -p 2222 user@server
  sudo ./scripts/harden-ssh.sh --finalize 2222   # закрывает 22
  ```
- `fail2ban` настроен на защиту SSH из коробки.
- IPv6 отключен по умолчанию (см. `/etc/sysctl.d/99-vpn-disable-ipv6.conf` для отката).
- Полный разбор того, что реально защищено, а что нет — **обязательно прочитайте
  [`docs/SECURITY.md`](docs/SECURITY.md)**.

| Домен (опционально, не обязателен — работает по IP) | ~$1/год |

## 11. Troubleshooting

**`docker compose up` падает / панель не открывается** — проверьте `docker compose logs wg-easy`,
убедитесь что порт 51820/udp не занят другим VPN-сервисом на сервере.

**Не удаётся подключиться к клиенту** — проверьте UFW: `sudo ufw status`, убедитесь что
`51820/udp` разрешён; проверьте, что `WG_HOST` в `.env` — правильный публичный IP.

**`update-routes.sh` падает с ошибкой источника** — оба источника (ipdeny.com, RIPE)
временно недоступны; списки останутся прежними (старая версия), повторите позже.

**Забыли пароль панели** — сгенерируйте новый хэш и обновите `.env`:
```sh
docker run --rm ghcr.io/wg-easy/wg-easy:14 wgpw 'новый-пароль'
# впишите результат в PASSWORD_HASH= (в одинарных кавычках!) в .env, затем:
docker compose up -d --force-recreate wg-easy
```

**Нужно полностью удалить VPN-сервер** — `sudo ./scripts/uninstall.sh` (бэкапы и
изменения SSH-порта не трогает, см. вывод скрипта).
